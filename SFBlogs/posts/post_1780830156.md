## 一、什么是"AI味"？

所谓的"AI味"，就是那种一眼就能看出是AI生成的设计作品，界面样式和内容风格都千篇一律。

### 典型AI味特征：

**视觉层面：**

- 配色死板：蓝紫渐变色用到吐

- 布局死板：首屏放大标题，下面三个卡片并排

- 字体死板：基本上就是 Inter、Roboto 等几种固定字体

- Emoji泛滥：满屏幕都是表情图标代替功能图标

- 柔和光晕效果，经常用在按钮、图标或重要文本周围

- 鼠标悬停按钮会有上浮动画和阴影效果

- 完美居中对齐，对称的"AI海报"布局

**文案层面：**

- 出现"智能化"、"赋能"、"效率提升"等空洞词汇

- 描述文案乍一看输出很多，但仔细琢磨说了等于没说

- 话术过于专业，没有"人味"，比较机械化的语言

**交互层面：**

- 没有细腻的动效和反馈

- 整体操作下来感觉很生硬

- 功能堆叠，逻辑缺乏连贯性

- 没有做到完全的响应式

### 为什么会有AI味？

核心原因就两个字：\*\*求稳\*\*。

AI的训练数据里，很多现代网站采用 Tailwind 样式库，默认主色调就是蓝紫色。AI在学习数亿行代码时，这些颜色出现的频率最高，于是AI就认为"现代化网站 ≈ 蓝紫色渐变"。

AI学会了一个生存法则：\*\*用最常见的 = 最不容易出错\*\*。

## 二、核心思路：从"请求者"变成"指挥官"

不要只说需求`给我做个网站`

而是要明确要求`用深灰色背景、手绘图标、不对称布局、拒绝蓝紫色`

**用强有力的约束条件，逼着AI偏离它的舒适区。**

## 三、视觉去AI味

### 3.1 色彩系统优化

**精简主色数量：**

- 高级感配色的铁律：主色≤1种，辅色≤2种，点缀色≤1种

- 邻近色搭配（色环60°内）如蓝+青+绿，通过明度变化建立层次

- 互补色如青橙（Teal & Orange）需控制面积比（主色占70%，点缀色≤10%）

**严禁使用AI默认色：**

- ❌ 纯蓝（#0000FF）或纯紫（#8000FF）

- ❌ 蓝紫色渐变（AI最爱）

- ✅ 优先选择具有品牌辨识度的中性色或高品质品牌色

- ✅ 确保对比度符合可读性标准

**校准色彩情绪：**

- 冷色调（蓝/绿/紫）传递科技、宁静感，适用于B2B或环保主题，但需提高明度（≥60）避免压抑

- 暖色调（红/橙/黄）激发食欲与活力，但饱和度应≤70以防刺眼

- 品牌调性优先于个人偏好

**设计禁止清单：**

```

❌ 紫色/靛蓝色渐变

❌ 纯平背景色（必须有噪点或渐变）

❌ 高深的专业名词和无意义的空话

❌ 过度使用 teal/orange 配色

❌ 不一致的白点 across 元素

❌ 浑浊的中性色

```

### 3.2 字体与排版

**禁止使用烂大街字体：**

- ❌ Inter、Roboto 等AI默认字体

- ✅ 使用具有个性的字体组合

**推荐字体组合：**

- 标题：\*\*Playfair Display\*\*（优雅衬线体）

- 正文：\*\*Bricolage Grotesque\*\*（现代无衬线体）

- 代码：\*\*JetBrains Mono\*\*（等宽字体）

- 其他组合：\*\*Cormorant Garamond\*\* + **Montserrat**

**排版规则：**

- 建立清晰的字号阶梯（如 text-4xl → text-sm）

- 标题与正文字号比大于2:1

- 使用变量字体轴（Variable Fonts）体现专业感

- 注意字间距（kerning）、基线（baseline）、光学对齐

**文案去AI味：**

- 避免"智能化"、"赋能"、"效率提升"等AI常用词

- 用口语化表达替换书面语

- 加入真实例子或个人经历

- 允许出现口语化断句

### 3.3 图标与素材

**严格禁止使用emoji：**

- ❌ 不要用 🐟4️⃣🐶 等表情代替功能图标

- ✅ 使用 Lucide、Heroicons 或 Tabler Icons 等高质量图标库

- ✅ 甚至生成独特的SVG矢量图标

**图案素材：**

- 替换默认图标，使用定制化图标/图片

- 可以去找开源可商用的图标网站

- 会设计的同学可以自己设计

### 3.4 布局与构图

**打破居中对称：**

- 使用非对称分栏

- 分镜式章节标题

- "破格"的布局

- 避免"Hero + 三卡片"的万能模板布局

**增加质感：**

- 引入体块叠合、模块化卡片

- 微妙的纸质纹理，而不是廉价的投影

- 多层渐变背景

- 微妙的网格或点阵图案

- 毛玻璃效果（backdrop-blur）

**背景处理：**

- ❌ 纯平背景色

- ✅ 多层渐变 + 细微纹理

- ✅ IDE灵感配色（如VSCode暗色主题）

- ✅ 通过分层创造深度感

### 3.5 引入大师级灵感池

与其让AI参考"现代风格"，不如让它跨界采样：

**平面设计：**

- Massimo Vignelli 的网格秩序

- Saul Bass 的极简符号

- Josef Müller-Brockmann 的瑞士风格

**建筑美学：**

- 扎哈·哈迪德的流动曲线

- 隈研吾的材质表达

- 安藤忠雄对光影与几何的克制

**工业设计：**

- Dieter Rams 的"少，但更好"

- 深泽直人的"无意识设计"

- Jony Ive 的细节偏执

**艺术风格：**

- Piet Mondrian 的几何抽象

- Matisse、Rothko 的色彩运用

- Yayoi Kusama 的重复图案

**转译而非临摹：**

- 仅借鉴气质与方法，禁止临摹具体作品

- 每位灵感需给出「灵感 → 网页实现」的转译说明

- 禁止复刻原作构图、配色、字体、标语或布局组合

## 四、约束优先提示词（Constraint-First Prompting）

### 4.1 为什么约束优先？

风格提示词（"让它有电影感"）很容易被模型遵循，但也同样容易被人们识别出来。

约束优先提示词更难伪造，因为它融入了真实的生产限制：网格、间距系统、排版意图和渠道要求。

**把AI当成初级设计师（给它明确的约束，要求多个方案，运行评审循环），而不是自动售货机。**

### 4.2 约束优先提示词模板

```

角色：资深艺术总监

交付物：[ASSET\_TYPE] 用于 [CHANNEL]，尺寸 [DIMENSIONS]

目标受众：[AUDIENCE]，场景 [CONTEXT]

品牌规则：

- 字体：[FONT\_1] 用于标题，[FONT\_2] 用于正文，字号比例 [TYPE\_SCALE]

- 颜色令牌：[PRIMARY\_HEX]，[ACCENT\_HEX]，中性色 [NEUTRAL\_SET]

- 布局：[GRID] 网格，边距 [MARGIN]，间距单位 [SPACING\_UNIT]

- 图像：[PHOTO\_STYLE\_CONSTRAINTS]（不要插画，不要手绘痕迹）

主题：[SUBJECT]，带有 [UNIQUE\_DETAIL] 以传达真实感

光照：[LIGHTING\_SETUP]，物理一致的阴影

相机：[LENS\_MM]，[APERTURE]，[ANGLE]

输出：8个变体，每个使用不同的构图规则（三分法、对角线、负空间、近景裁剪）

避免：通用海报布局、居中主体、随机伪文本、过度平滑的表面

```

### 4.3 反向提示词技巧

不要只说"要什么"，更要说"不要什么"。

**设计禁止清单示例：**

```

❌ 紫色/靛蓝色渐变

❌ 纯平背景色（必须有噪点或渐变）

❌ Hero + 三卡片布局

❌ 完美居中对齐

❌ Emoji 作为功能图标

❌ 通用居中英雄布局

❌ 随机伪文本

❌ 过度平滑的表面

❌ 塑料般的光滑感

❌ 蜡状渐变

❌ 无孔表面

```

### 4.4 让AI参考真实网站

**4种具体做法：**

1. **直接让AI读取网页**（使用 Cursor、Claude Code 或 Firecrawl MCP）：

```

请访问 [example.com](http://example.com)，提取它的配色方案、字体选择和布局结构，然后生成类似风格的网站。

```

2. **截图提供给AI**（如果AI支持图片理解）：搭配文字让AI还原网站更准确

3. **截图转代码工具**（如 Screenshot to Code）：把喜欢的网站截图上传，直接转成代码，再喂给AI

4. **套用现成模板**：

- [HTML5 UP](<https://html5up.net/)：免费响应式网站合集，极简风格>

- [WordPress 官方主题库](<https://cn.wordpress.org/themes/)：1万多个免费主题>

- [Start Bootstrap](<https://startbootstrap.com/)：Bootstrap生态免费网站模板库>

- [Colorlib](<https://colorlib.com/)：设计精美的免费网站模板>

## 五、文案去AI味

### 5.1 AI文案常见问题

- 过度正式用语

- 缺乏具体细节

- 万能梗滥用（如每章都出现"糖炒栗子"）

- 结构僵化（"首先、其次、最后"）

- 语言正式、缺乏情感

### 5.2 去AI文案技巧

**角色绑定法：**

给AI一个具体的"人设锚点"，而非笼统的"写一篇文章"。

例如：

- "假设你是一位刚解决线上BUG的程序员，正在给团队写复盘文档，要求用'凌晨三点踩坑'的真实语气"

- "加入'当时误把日志级别写成INFO，导致排查半小时'的具体细节"

**反结构指令：**

- 禁止使用序号："不要用'第一、第二'，改用'先说个容易踩的坑'"

- 强制插入"思维断点"："在第二段中间加入一句'等等，这里有个矛盾点'"

- 限定"非完整表达"："允许出现3处以上的口语化断句"

**缺陷注入法：**

- 故意保留错误："在代码示例中加入1处非致命错误"

- 强化情感倾向："明确表达你对某个方案的偏好"

- 插入生活碎片："在技术分析中加入一句无关但真实的细节"

### 5.3 通用去痕迹提示词

```

请以人类真实表达的方式输出内容，避免使用以下AI特征：

- 避免使用"值得注意的是"、"综上所述"、"首先...其次...最后"等套路化表达

- 不要使用过于正式或书面化的语言

- 加入具体的例子或个人经历

- 允许观点有主观性

- 使用口语化过渡词

```

## 六、交互与动效去AI味

### 6.1 动效原则

**拒绝浮夸：**

- 设置 200–300ms 的入场动画或滚动反馈

- 必须支持 `prefers-reduced-motion` 静态回退

- 不要过度动画，保持高级感

**细腻的动效和反馈：**

- CSS transitions 用于悬停状态

- 交错页面加载动画

- 按钮/卡片上的微交互

- 尊重 `prefers-reduced-motion` 设置

### 6.2 响应式设计

- 必须兼容 PC + 平板 + 手机

- 在移动端，布局不能错乱，图标不能乱飞

- 初始化提示词到位的话，优秀的AI IDE能做到适配

## 七、设计优先开发流程

### 7.1 不要上来就让AI梭哈整个项目

**传统做法（错误）：**

```

帮我做一个包含用户系统、后台管理的完整SaaS平台

```

然后AI哗啦哗啦给你生成几十个文件，结果做完后发现页面风格不对，重新返工，浪费Tokens。

**推荐做法（正确）：**

1. 先让AI做个前端网站Demo，纯静态页面就好

2. 对设计满意了之后，再让AI基于Demo代码，用同样的风格开发完整项目

### 7.2 6阶段"草稿到成品"检查清单

```

1) 意图锁定：定义受众、渠道、品牌规则和约束（网格、字号比例、颜色令牌）

2) 草稿生成：生成6-12个选项，控制变化（相机、镜头、光照、构图）

3) AI痕迹扫描：审核AI artifacts（微细节、阴影逻辑、边缘、反射、解剖、排版）

4) 结构编辑：修复透视、几何、布局、层次、裁剪

5) 材质编辑：恢复纹理、噪点、镜头特征、真实的不完美（不是假的手工感）

6) 品牌审核：排版系统、间距系统、色彩管理、导出规范、无障碍检查

```

### 7.3 90秒"AI痕迹扫描"清单

```

光照：阴影一致吗？高光匹配材质吗？一个主光源方向吗？

几何：透视线条正确收敛吗？物体比例可信吗？

边缘：有光晕、切割毛边、过度锐化轮廓、融化细节吗？

微细节：有随机符号、假文字、重复图案、不可能的接缝吗？

反射：镜像逻辑正确吗？反射匹配环境和相机角度吗？

皮肤/材质：塑料般光滑？蜡状渐变？无孔表面？

排版：字间距、基线、光学对齐、真实字体使用、一致的字号比例？

构图：通用居中英雄、对称"AI海报"布局、没有叙事焦点路径？

颜色：浑浊中性色、过度青橙、不一致的白点？

品牌：遵守间距令牌、logo安全空间、组件规则吗？

```

## 八、完整提示词示例

### 8.1 产品介绍网页去AI味提示词

```

你现在是一名资深的前端开发工程师 + 产品设计师，请帮我从0到1编写一个完整的「XXX产品介绍网页」。

【一、项目目标】

创建一个高质量、具有商业级展示效果的产品介绍网页，用于展示"XXX产品"，要求页面视觉高级、结构清晰、信息完整，适合用于官网或宣传落地页。

【二、页面整体要求】

1. 页面类型：单页（Landing Page）

2. 风格要求：简洁 / 高级 / 现代科技感（可根据产品类型自动调整）

3. 响应式设计：必须兼容 PC + 平板 + 手机

4. 技术要求：

- 使用 HTML + CSS + JavaScript

- 代码结构清晰、可直接运行

- 样式不要杂乱，避免重复代码

5. 动效要求：

- 适当加入滚动动画（如淡入、上移）

- 按钮、卡片需有 hover 效果

- 不要过度动画，保持高级感

【三、设计约束（去AI味关键）】

1. 色彩：

- 禁止使用蓝紫色渐变

- 主色不超过1种，辅色不超过2种

- 使用CSS变量定义颜色系统

- 确保对比度符合可读性标准

2. 字体：

- 禁止使用 Inter、Roboto

- 标题使用 Playfair Display 或其他有特色的衬线体

- 正文使用 Bricolage Grotesque 或 Montserrat

- 代码使用 JetBrains Mono

3. 图标：

- 严格禁止使用 emoji

- 使用 Lucide、Heroicons 或 Tabler Icons

4. 布局：

- 打破居中对称，使用非对称分栏

- 避免"Hero + 三卡片"的万能模板

- 增加质感（微妙纹理、体块叠合）

5. 背景：

- 禁止纯平背景色

- 使用多层渐变 + 细微纹理

- 可加入网格或点阵图案

【四、页面结构（必须完整实现）】

1. 顶部导航栏（Header）- 固定在顶部（sticky）

2. 首屏Banner（Hero Section）- 非对称布局

3. 产品介绍（About Section）- 图文排版

4. 核心功能（Features Section）- 4~6个功能点

5. 产品优势（Advantages Section）- 对比传统方案

6. 使用场景（Use Cases）- 卡片式布局

7. 用户评价 / 案例（Testimonials）- 2~3条模拟评价

8. 价格/套餐（Pricing，可选但建议）

9. CTA行动区（Call To Action）

10. 页脚（Footer）

【五、内容生成要求】

1. 所有文案必须完整生成，不要留"待补充"

2. "XXX产品"相关内容要合理虚构，但要逻辑真实、专业

3. 文案风格偏产品经理 + 市场营销风格

4. 避免"智能化"、"赋能"、"效率提升"等AI常用词

5. 使用口语化、有温度的表达

【六、输出要求】

1. 输出完整代码（HTML + CSS + JS）

2. 所有代码写在一个文件中

3. 可直接复制运行

4. 代码中添加必要注释

【七、禁止事项】

- 不要只给结构说明，必须输出完整代码

- 不要省略模块

- 不要使用过时或低质量UI

- 不要使用蓝紫色渐变

- 不要使用emoji作为图标

- 不要完美居中对齐

请直接开始生成最终网页代码。

```

### 8.2 用AI优化提示词

**第一步：用AI出提示词**

```

你是一位专业的提示词工程师（Prompt Engineer），擅长将模糊、不完整的需求，优化为可以让 AI Agent 高质量执行的提示词。

我会给你一段"原始提示词"，它存在表达不清、细节不足、结构混乱的问题。

你的任务是：

1. 完全理解我的真实意图（不要只做表面润色）

2. 补全缺失的关键细节（包括功能、逻辑、约束、边界情况）

3. 重构提示词结构，让 AI Agent 更容易理解和执行

4. 明确输出格式（非常重要）

5. 避免歧义，让结果尽可能稳定、可复现

优化要求：

- 提示词必须结构清晰（建议使用分点或模块化）

- 语言必须具体、明确，避免模糊词（如"好看一点""优化一下"）
```

## 九、Anti-AI设计趋势（2026）

### 9.1 拥抱有意不完美（Wabi-Sabi）

在算法完美的数字景观中，不完美已成为终极奢侈品。

**不完美的功能：**

- **真实性信号：** 粗糙边缘和手工元素传达人类参与和创意意图

- **情感共鸣：** 不完美元素创造温暖与个性

- **视觉区分：** 在AI生成的同质化海洋中脱颖而出

- **建立信任：** 受众越来越将完美与人工和操纵联系起来

**实施技巧：**

- 手绘线条、注释和箭头添加到 polished AI图形

- 纸张纹理、画布或织物纹理叠加在数字构图上（10-30%透明度）

- 扫描仪线条、复印机噪点或打印套准误差

- 剪贴美学，参考物理艺术制作过程

- 手写字体、定制字母和书法与数字字体混合

### 9.2 复古与粗野主义影响

**复古影响：**

- 90年代网页美学：块状布局、原色、故意原始的图形

- Y2K未来主义：金属效果、镜头光晕、光泽3D元素

- 粗野主义排版：厚重、几何无衬线体和暴露的结构

- 复古计算：CRT扫描线、像素化、终端风格界面

- 朋克/杂志文化：剪贴拼贴、复印机美学、DIY能量

### 9.3 触觉灵感与材质

**低成本方案（$0-50）：**

- 使用高分辨率扫描的撕裂纸张、亚麻布或织物作为叠加层

- 以5-15%透明度使用 multiply 混合模式

- Unsplash 和 Pixabay 有免费纹理包

**中等预算（$50-500）：**

- 在排版中引入"噪点"

- 使用模拟墨水扩散或活版印刷凹陷的字体
