## 1. 安装宝塔面板（如果还没有）

如果你已经安装了宝塔面板，跳过此步骤。

### 1.1 通过 SSH 连接到你的服务器

- **Windows 用户**：下载并打开 [PuTTY](https://www.putty.org/)，或者使用 Windows 自带的终端（按 `Win + R`，输入 `cmd`，回车），然后输入：
  ```
  ssh root@你的服务器IP地址
  ```
  例如：`ssh root@123.45.67.89`

- **Mac 用户**：打开"终端"应用，输入同样的命令

> 💡 **什么是服务器 IP 地址？** 就是你买服务器时，服务商给你的那个 IP 地址，类似 `123.45.67.89` 的一串数字。

输入后按回车，如果提示 `Are you sure you want to continue connecting?`，输入 `yes` 并回车。然后输入你的服务器密码（输入时屏幕上不会显示任何字符，这是正常的），按回车即可登录。

### 1.2 安装宝塔面板

登录服务器后，根据你的系统选择对应的命令：

**Ubuntu / Debian 系统**（如果你不确定，选这个）：
```bash
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh ed8484bec
```

**CentOS 系统**：
```bash
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec
```

安装过程中会问你 `Do you want to install Bt-Panel to the /www directory now?`，输入 `y` 并回车。

等待安装完成（大约 2-5 分钟），安装完成后屏幕上会显示：
- **面板访问地址**：类似 `http://123.45.67.89:8888/xxxx`
- **用户名**：一串字符
- **密码**：一串字符

> ⚠️ **重要**：把这些信息复制保存下来！后面要用！

## 2. 登录宝塔面板

1. 打开你电脑上的浏览器（Chrome、Edge 等都可以）
2. 在地址栏粘贴刚才保存的**面板访问地址**，按回车
3. 输入刚才保存的**用户名**和**密码**
4. 点击"登录"

### 2.1 绑定宝塔账号（首次登录）

首次登录会要求绑定宝塔官方账号：
- 如果你已有账号，直接登录
- 如果没有，点击注册一个（免费的），然后绑定

### 2.2 推荐安装套件

登录后会弹出一个窗口，让你选择安装套件。选择 **LNMP（推荐）**：
- **Nginx**：选择最新版本（默认即可）
- **MySQL**：可以不安装（本项目不需要 MySQL），或者选默认版本也行
- **PHP**：可以不安装（本项目不需要 PHP），或者选默认版本也行
- **phpMyAdmin**：可以不安装

点击"一键安装"，等待安装完成（大约 5-15 分钟）。

>  如果你不需要 MySQL 和 PHP，可以在弹窗中把它们取消勾选，只安装 Nginx 即可。

## 3. 安装必要软件

本项目需要以下软件：
- **Node.js**（版本管理器，v18+）
- **Python**（3.10+）
- **MongoDB**（6.0+）
- **PM2**（进程管理器，用于让服务在后台持续运行）

### 3.1 安装 Node.js

1. 在宝塔面板左侧菜单，点击"软件商店"
2. 在搜索框中输入 `Node.js版本管理器`
3. 找到"Node.js版本管理器"，点击"安装"
4. 等待安装完成
5. 安装完成后，点击"设置"
6. 在弹出的窗口中，找到 Node.js 版本列表，选择 **v18.x.x**（选最新的 18.x 版本），点击"安装"
7. 等待 Node.js 安装完成

> 💡 **什么是 Node.js？** 它是一个 JavaScript 运行环境，我们的博客前端和后台管理器都是用它运行的。

### 3.2 安装 Python

1. 在宝塔面板左侧菜单，点击"软件商店"
2. 搜索 `Python管理器`
3. 找到"Python管理器"，点击"安装"
4. 等待安装完成
5. 安装完成后，点击"设置"
6. 点击"版本管理"，选择 **Python 3.10** 或更高版本，点击"安装"
7. 等待安装完成

> 💡 如果你的系统已经自带了 Python 3.10+，可以跳过这步。在终端输入 `python3 --version` 可以查看版本。

### 3.3 安装 MongoDB

1. 在宝塔面板左侧菜单，点击"软件商店"
2. 搜索 `MongoDB`
3. 找到"MongoDB"，点击"安装"
4. 选择最新版本，点击"提交"
5. 等待安装完成（大约 2-5 分钟）
6. 安装完成后，确保 MongoDB 服务状态为"运行中"

> 💡 **什么是 MongoDB？** 它是数据库，用来存储博客的评论、访客说说、公告和访问记录数据。

### 3.4 安装 PM2

1. 在宝塔面板左侧菜单，点击"终端"
2. 在终端中输入以下命令并回车：
   ```bash
   npm install -g pm2
   ```
3. 等待安装完成（大约 30 秒）

> 💡 **什么是 PM2？** 它是一个进程管理器，可以让我们的博客服务在后台持续运行，即使关闭终端也不会停止。

## 4. 上传项目文件

### 4.1 创建项目目录

1. 在宝塔面板左侧菜单，点击"文件"
2. 进入 `/www/wwwroot` 目录（点击左侧的目录树，找到 www → wwwroot）
3. 点击上方的"新建目录"按钮
4. 输入目录名 `NoWin_Blog`，点击"确定"

### 4.2 上传项目文件

**方式一：从本地上传（推荐新手使用）**

1. 在你的电脑上，将整个 `NoWin_Blog` 项目文件夹压缩成 `.zip` 文件
   - Windows：右键点击 `NoWin_Blog` 文件夹 → "压缩为 ZIP 文件"
   - Mac：右键点击 → "压缩"
2. 在宝塔面板的文件管理器中，进入 `/www/wwwroot/NoWin_Blog` 目录
3. 点击上方的"上传"按钮
4. 将刚才的 `.zip` 文件拖入上传区域，等待上传完成
5. 上传完成后，选中该 `.zip` 文件，点击"解压"，选择解压到当前目录
6. 解压后你会看到 `NWBlogs` 和 `my-blog-manager` 两个文件夹

**方式二：通过 Git 克隆（如果你会用 Git）**

1. 在宝塔面板左侧菜单，点击"终端"
2. 输入以下命令：
   ```bash
   cd /www/wwwroot/NoWin_Blog
   git clone https://github.com/你的用户名/NoWin_Blog.git .
   ```

### 4.3 确认目录结构

上传完成后，在宝塔面板的文件管理器中，确认目录结构如下：

```
/www/wwwroot/NoWin_Blog/
├── NWBlogs/              ← 博客前端
── my-blog-manager/      ← 后台管理器
├── Screenshot/           ← 页面截图
├── README.md
└── 如何用服务器的宝塔面板将博客项目部署到你的服务器中.md
```

> ⚠️ **重要**：确保 `NWBlogs` 和 `my-blog-manager` 这两个文件夹直接在 `NoWin_Blog` 目录下，而不是嵌套了一层。

## 5. 安装项目依赖

### 5.1 安装前端依赖

1. 在宝塔面板左侧菜单，点击"终端"
2. 依次输入以下命令（每输入一行按一次回车，等待执行完毕再输入下一行）：

   ```bash
   cd /www/wwwroot/NoWin_Blog/NWBlogs
   npm install
   ```

3. 等待安装完成（大约 1-3 分钟），看到出现新的命令提示符 `$` 或 `#` 就表示完成了

> ⚠️ 如果报错 `npm: command not found`，说明 Node.js 还没安装好或没生效。回到 [3.1 安装 Node.js](#31-安装-nodejs) 检查。

### 5.2 安装后台管理器依赖

继续在终端中输入：

```bash
cd /www/wwwroot/NoWin_Blog/my-blog-manager
npm install
```

等待安装完成。

### 5.3 安装 Python 依赖

继续在终端中输入：

```bash
pip3 install fastapi uvicorn python-multipart PyYAML markdown markdownify httpx requests pymongo
```

等待安装完成（大约 30 秒-1 分钟）。

> ⚠️ 如果报错 `pip3: command not found`，试试 `pip install` 代替 `pip3 install`。如果还是不行，试试 `python3 -m pip install fastapi uvicorn python-multipart PyYAML markdown markdownify httpx requests pymongo`。

## 6. 构建博客前端

前端项目需要先"构建"才能在生产环境中运行。

1. 在终端中输入：
   ```bash
   cd /www/wwwroot/NoWin_Blog/NWBlogs
   npm run build
   ```

2. 等待构建完成（大约 2-5 分钟），看到类似以下输出就表示成功：
   ```
   ✓ Compiled successfully
   ```

3. 构建完成后，确认 `.next/standalone` 目录存在：
   ```bash
   ls -la .next/standalone
   ```
   应该能看到 `server.js` 等文件。

> ️ 如果构建过程中报错，通常是内存不足导致的。1GB 内存的服务器可能需要添加虚拟内存（swap），参见 [常见问题排查](#16-常见问题排查)。

## 7. 配置 MongoDB 数据库

### 7.1 启动 MongoDB

1. 在宝塔面板左侧菜单，点击"软件商店"
2. 找到"MongoDB"，确认状态为"运行中"
3. 如果没有运行，点击"启动"

### 7.2 验证 MongoDB 连接

在终端中输入：

```bash
mongosh --eval "db.runCommand({ ping: 1 })"
```

如果看到 `{ ok: 1 }` 就表示 MongoDB 运行正常。

> 💡 如果报错 `mongosh: command not found`，试试 `mongo --eval "db.runCommand({ ping: 1 })"`。

### 7.3 MongoDB 无需额外配置

本项目使用的 MongoDB 默认配置即可，数据库名为 `nowin_blog`，会在首次运行时自动创建。评论、访客说说、公告和访问记录的集合都会自动生成，你不需要手动创建数据库或表。

## 8. 使用 PM2 启动后端服务

### 8.1 创建后端启动脚本

1. 在宝塔面板的文件管理器中，进入 `/www/wwwroot/NoWin_Blog/my-blog-manager` 目录
2. 点击"新建文件"，文件名输入 `start_backend.sh`
3. 选中该文件，点击"编辑"，粘贴以下内容：

   ```bash
   #!/bin/bash
   cd /www/wwwroot/NoWin_Blog/my-blog-manager
   export MONGO_URI="mongodb://localhost:27017"
   export MONGO_DB_NAME="nowin_blog"
   python3 -c "from cms_core.main import app; import uvicorn; uvicorn.run(app, host='0.0.0.0', port=8765)"
   ```

4. 点击"保存"

### 8.2 用 PM2 启动后端

在终端中输入：

```bash
cd /www/wwwroot/NoWin_Blog/my-blog-manager
pm2 start start_backend.sh --name nowin-blog-api
```

看到类似以下输出就表示启动成功：

```
┌────┬──────────────────┬─────────────┬─────────┐
│ id │ name             │ status      │ cpu     │
├────┼──────────────────┼─────────────┼─────────┤
│ 0  │ nowin-blog-api   │ online      │ 0%      │
└────┴──────────────────┴─────────────┴─────────┘
```

> ⚠️ 如果 `status` 显示 `errored` 或 `stopped`，说明启动失败。输入 `pm2 logs nowin-blog-api` 查看错误日志。

### 8.3 验证后端是否运行

在终端中输入：

```bash
curl http://127.0.0.1:8765/api/status
```

如果看到 `{"status":"online","message":"中枢神经已连接"}` 就表示后端运行正常。

## 9. 使用 PM2 启动前端服务

### 9.1 创建前端启动脚本

1. 在宝塔面板的文件管理器中，进入 `/www/wwwroot/NoWin_Blog/NWBlogs` 目录
2. 点击"新建文件"，文件名输入 `start_frontend.sh`
3. 选中该文件，点击"编辑"，粘贴以下内容：

   ```bash
   #!/bin/bash
   cd /www/wwwroot/NoWin_Blog/NWBlogs/.next/standalone
   export HOSTNAME="0.0.0.0"
   export PORT="3000"
   export CMS_BACKEND_URL="http://127.0.0.1:8765"
   export MONGO_URI="mongodb://localhost:27017"
   export MONGO_DB_NAME="nowin_blog"
   node server.js
   ```

4. 点击"保存"

### 9.2 复制静态资源到 standalone 目录

构建后的 standalone 目录不包含静态资源，需要手动复制：

在终端中输入：

```bash
cd /www/wwwroot/NoWin_Blog/NWBlogs
cp -r .next/static .next/standalone/.next/
cp -r public .next/standalone/
```

> ⚠️ 这一步很重要！如果不复制，页面样式会丢失。

### 9.3 用 PM2 启动前端

在终端中输入：

```bash
cd /www/wwwroot/NoWin_Blog/NWBlogs
pm2 start start_frontend.sh --name nowin-blog-web
```

### 9.4 验证前端是否运行

在终端中输入：

```bash
curl http://127.0.0.1:3000
```

如果看到一大段 HTML 代码输出，就表示前端运行正常。

### 9.5 设置 PM2 开机自启

在终端中输入：

```bash
pm2 save
pm2 startup
```

> 💡 这一步会让 PM2 记住当前运行的服务，并在服务器重启后自动恢复运行。如果 `pm2 startup` 输出了一条以 `sudo` 开头的命令，请复制那条命令并执行。

## 10. 配置 Nginx 反向代理

### 10.1 添加站点

1. 在宝塔面板左侧菜单，点击"网站"
2. 点击"添加站点"
3. 填写信息：
   - **域名**：输入你的域名（如 `blog.example.com`），如果没有域名，输入你的服务器 IP
   - **根目录**：保持默认即可（本项目不需要）
   - **PHP版本**：选择"纯静态"
   - **数据库**：选择"不创建"
4. 点击"提交"

### 10.2 配置反向代理

1. 在网站列表中，找到刚才添加的站点，点击站点名称
2. 点击左侧的"反向代理"选项卡
3. 点击"添加反向代理"
4. 填写信息：
   - **代理名称**：输入 `blog`
   - **目标URL**：输入 `http://127.0.0.1:3000`
   - **发送域名**：输入 `$host`
5. 点击"提交"

### 10.3 配置 API 代理（重要）

上一步只配置了前端的代理，还需要配置后端 API 的代理。

1. 在网站列表中，点击该站点的"设置"
2. 点击左侧的"配置文件"选项卡
3. 在 `location /` 之前，添加以下配置：

   ```nginx
   location /api/ {
       proxy_pass http://127.0.0.1:8765/api/;
       proxy_set_header Host $host;
       proxy_set_header X-Real-IP $remote_addr;
       proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_connect_timeout 60s;
       proxy_read_timeout 120s;
       proxy_send_timeout 60s;
   }
   ```

4. 点击"保存"

> 💡 **这段配置的作用**：当访问你的博客的 `/api/` 路径时，Nginx 会把请求转发到 Python 后端（8765 端口），其他路径则转发到前端（3000 端口）。

## 11. 开放防火墙端口

### 11.1 在宝塔面板中放行端口

1. 在宝塔面板左侧菜单，点击"安全"
2. 在"放行端口"输入框中，输入 `80`，点击"放行"
3. 同样放行以下端口：
   - `443`（HTTPS）
   - `8888`（宝塔面板，默认已放行）

> 💡 前端（3000 端口）和后端（8765 端口）不需要对外放行，因为 Nginx 已经通过反向代理将 80 端口的请求转发到了这两个端口。直接暴露 3000 和 8765 端口会有安全风险。

### 11.2 在云服务商控制台放行端口

除了宝塔面板的防火墙，你还需要在云服务商（阿里云、腾讯云、华为云等）的控制台中放行端口：

1. 登录你的云服务商控制台
2. 找到"安全组"或"防火墙规则"
3. 添加入站规则：
   - 端口 `80`，协议 TCP，来源 `0.0.0.0/0`
   - 端口 `443`，协议 TCP，来源 `0.0.0.0/0`

> 💡 **不同云服务商的位置不同**：
> - **阿里云**：云服务器 ECS → 安全组 → 配置规则 → 添加安全组规则
> - **腾讯云**：轻量应用服务器 → 防火墙 → 添加规则
> - **华为云**：弹性云服务器 → 安全组 → 入方向规则 → 添加规则

## 12. 验证部署是否成功

### 12.1 访问博客前端

在浏览器中输入：

- 如果有域名：`http://你的域名`
- 如果没有域名：`http://你的服务器IP地址`

如果看到博客首页，恭喜你，部署成功！

### 12.2 访问后台管理器

在浏览器中输入：

- 如果有域名：`http://你的域名:3001`
- 如果没有域名：`http://你的服务器IP地址:3001`

> ⚠️ 如果要访问后台管理器，你还需要额外部署 my-blog-manager 的前端。参见下面的补充步骤。

### 12.3 补充：部署后台管理器前端

如果你需要通过浏览器访问后台管理器，还需要额外构建和启动它：

1. 构建后台管理器前端：
   ```bash
   cd /www/wwwroot/NoWin_Blog/my-blog-manager
   npm run build
   ```

2. 复制静态资源：
   ```bash
   cd /www/wwwroot/NoWin_Blog/my-blog-manager
   cp -r .next/static .next/standalone/.next/
   cp -r public .next/standalone/
   ```

3. 创建启动脚本。在 `/www/wwwroot/NoWin_Blog/my-blog-manager` 目录下新建 `start_manager.sh`，内容如下：
   ```bash
   #!/bin/bash
   cd /www/wwwroot/NoWin_Blog/my-blog-manager/.next/standalone
   export HOSTNAME="0.0.0.0"
   export PORT="3001"
   export CMS_BACKEND_URL="http://127.0.0.1:8765"
   export MONGO_URI="mongodb://localhost:27017"
   export MONGO_DB_NAME="nowin_blog"
   node server.js
   ```

4. 用 PM2 启动：
   ```bash
   cd /www/wwwroot/NoWin_Blog/my-blog-manager
   pm2 start start_manager.sh --name nowin-blog-manager
   pm2 save
   ```

5. 在宝塔面板"安全"中放行 `3001` 端口，并在云服务商安全组中也放行该端口

6. 访问 `http://你的服务器IP:3001` 即可进入后台管理器

> ⚠️ 3001 端口对外暴露有安全风险，建议只通过 IP 白名单或 VPN 访问。更安全的做法是为后台管理器也配置一个 Nginx 反向代理，并加上访问密码。

## 13. 配置域名（可选）

如果你有域名，可以绑定到博客上，这样访问更方便。

### 13.1 添加域名解析

1. 登录你的域名服务商（阿里云、腾讯云、Cloudflare 等）
2. 进入域名管理 → DNS 解析
3. 添加一条 A 记录：
   - **主机记录**：输入你想用的子域名，如 `blog`
   - **记录类型**：A
   - **记录值**：你的服务器 IP 地址
4. 等待 DNS 生效（通常几分钟到几小时不等）

### 13.2 在宝塔面板中绑定域名

1. 在宝塔面板左侧菜单，点击"网站"
2. 找到你的站点，点击"设置"
3. 在左侧点击"域名管理"
4. 在输入框中输入你的域名（如 `blog.example.com`），点击"添加"

## 14. 配置 SSL 证书（可选但推荐）

配置 SSL 证书后，你的博客可以通过 `https://` 访问，更安全。

### 14.1 申请免费 SSL 证书

1. 在宝塔面板中，点击你的站点"设置"
2. 点击左侧的"SSL"选项卡
3. 选择"Let's Encrypt"选项卡
4. 勾选你的域名
5. 点击"申请"
6. 等待申请完成（通常几秒钟）

### 14.2 开启强制 HTTPS

1. 申请成功后，点击"强制HTTPS"开关，将其打开
2. 这样所有 `http://` 的访问都会自动跳转到 `https://`

## 16. 常见问题排查

### 16.1 前端页面空白或样式丢失

**原因**：构建后没有复制静态资源到 standalone 目录。

**解决方法**：
```bash
cd /www/wwwroot/NoWin_Blog/NWBlogs
cp -r .next/static .next/standalone/.next/
cp -r public .next/standalone/
pm2 restart nowin-blog-web
```

### 16.2 后端启动失败

**原因**：Python 依赖未安装或 MongoDB 未启动。

**解决方法**：
```bash
# 检查 Python 依赖
pip3 install fastapi uvicorn python-multipart PyYAML markdown markdownify httpx requests pymongo

# 检查 MongoDB 是否运行
mongosh --eval "db.runCommand({ ping: 1 })"

# 查看后端日志
pm2 logs nowin-blog-api --lines 50
```

### 16.3 构建时内存不足（OOM）

**原因**：服务器内存太小（如 1GB），Next.js 构建需要较多内存。

**解决方法**：添加虚拟内存（swap）：
```bash
# 创建 2GB 的 swap 文件
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 16.4 评论或访客说说无法提交

**原因**：MongoDB 连接失败或后端 API 未正常运行。

**解决方法**：
```bash
# 检查后端是否运行
curl http://127.0.0.1:8765/api/status

# 检查 MongoDB 是否运行
mongosh --eval "db.runCommand({ ping: 1 })"

# 检查 Nginx 反向代理配置是否正确
cat /www/server/panel/vhost/nginx/你的域名.conf
```

### 16.5 文章发布后前台不显示

**原因**：ISR 缓存未过期，需要等待 60 秒或手动刷新。

**解决方法**：
- 等待 60 秒后刷新页面（文章页 ISR 重新验证时间为 60 秒）
- 或者重启前端服务：`pm2 restart nowin-blog-web`

### 16.6 说说发布后前台不显示

**原因**：后台发布的说说需要同步到前端目录。

**解决方法**：
- 在后台管理器中发布说说后，系统会自动同步到 `NWBlogs/moments/` 目录
- 如果未同步，检查后端日志：`pm2 logs nowin-blog-api --lines 50`

### 16.7 端口被占用

**原因**：3000、3001 或 8765 端口已被其他程序占用。

**解决方法**：
```bash
# 查看端口占用
lsof -i :3000
lsof -i :3001
lsof -i :8765

# 杀死占用端口的进程
kill -9 <PID>

# 或者修改端口（在启动脚本中修改 PORT 环境变量）
```

### 16.8 Nginx 反向代理 502 错误

**原因**：Nginx 无法连接到后端服务。

**解决方法**：
```bash
# 检查前端和后端是否都在运行
pm2 list

# 检查 Nginx 配置
nginx -t

# 重启 Nginx
systemctl restart nginx

# 检查防火墙
ufw status
```
