## 1. 服务器准备与宝塔面板安装

### 1.1 通过 SSH 连接服务器

- **Windows 用户**：打开终端（`Win + R` → 输入 `cmd`），执行：
  ```
  ssh root@你的服务器IP地址
  ```
- **Mac 用户**：打开"终端"应用，执行同样命令

首次连接提示 `Are you sure you want to continue connecting?` 时输入 `yes`，然后输入服务器密码登录。

### 1.2 安装宝塔面板

根据系统选择安装命令：

**Ubuntu / Debian 系统**：
```bash
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh ed8484bec
```

**CentOS 系统**：
```bash
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec
```

安装过程中输入 `y` 确认。安装完成后屏幕会显示面板访问地址、用户名和密码，务必保存。

## 2. 登录宝塔面板并安装基础软件

### 2.1 登录面板

1. 浏览器打开面板访问地址
2. 输入用户名和密码登录
3. 首次登录需绑定宝塔官方账号（免费注册）

### 2.2 安装 LNMP 套件

登录后弹出套件选择窗口，选择 **LNMP**：
- **Nginx**：安装最新版本
- **MySQL**：可不安装（本项目不需要）
- **PHP**：可不安装（本项目不需要）

点击"一键安装"等待完成。

## 3. 安装运行环境（Node.js / Python / MongoDB / PM2）

### 3.1 安装 Node.js

1. 宝塔面板 → 软件商店 → 搜索 `Node.js版本管理器` → 安装
2. 安装完成后点击"设置" → 选择 **v18.x.x** → 安装
3. 等待安装完成

### 3.2 安装 Python

1. 宝塔面板 → 软件商店 → 搜索 `Python管理器` → 安装
2. 安装完成后点击"设置" → 版本管理 → 选择 **Python 3.10+** → 安装

> 如果系统已自带 Python 3.10+（终端执行 `python3 --version` 查看），可跳过此步。

### 3.3 安装 MongoDB

1. 宝塔面板 → 软件商店 → 搜索 `MongoDB` → 安装
2. 安装完成后确认服务状态为"运行中"

### 3.4 安装 PM2

在宝塔终端中执行：

```bash
npm install -g pm2
```

## 8. 配置安全环境变量

> ⚠️ **生产环境必须执行此步骤**，否则存在安全风险。

### 8.1 生成 JWT 密钥

在终端中执行：

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

复制输出的 64 位字符串，这就是你的 `CMS_SECRET_KEY`。

### 8.2 创建环境变量文件

在 `/www/wwwroot/SilentFall_Blog/my-blog-manager` 目录下新建 `.env` 文件，内容如下：

```bash
# MongoDB 配置
MONGO_URI=mongodb://localhost:27017
MONGO_DB_NAME=silentfall_blog

# JWT 安全密钥（替换为你刚才生成的密钥）
CMS_SECRET_KEY=你刚才生成的64位字符串

# CORS 允许的来源（替换为你的实际域名）
CMS_ALLOWED_ORIGINS=http://你的域名,http://你的服务器IP

# JWT Token 有效期（小时）
CMS_TOKEN_EXPIRE_HOURS=24

# 登录速率限制
CMS_LOGIN_WINDOW_SECONDS=300
CMS_LOGIN_MAX_ATTEMPTS=5
```

> ⚠️ `CMS_SECRET_KEY` 必须设置，否则每次重启后端后所有已登录用户的 Token 会失效。`CMS_ALLOWED_ORIGINS` 必须设置为你实际的前端域名，不要使用默认的 `*`。

## 9. 使用 PM2 启动后端服务

### 9.1 创建启动脚本

在 `/www/wwwroot/SilentFall_Blog/my-blog-manager` 目录下新建 `start_backend.sh`：

```bash
#!/bin/bash
cd /www/wwwroot/SilentFall_Blog/my-blog-manager

# 加载 .env 文件中的环境变量
set -a
source .env
set +a

python3 -c "from cms_core.main import app; import uvicorn; uvicorn.run(app, host='0.0.0.0', port=8765)"
```

### 9.2 启动后端

```bash
cd /www/wwwroot/SilentFall_Blog/my-blog-manager
pm2 start start_backend.sh --name silentfall-blog-api
```

### 9.3 验证

```bash
curl http://127.0.0.1:8765/api/status
```

返回 `{"status":"online","message":"中枢神经已连接"}` 表示正常。

## 12. 配置 Nginx 反向代理

### 12.1 添加站点

1. 宝塔面板 → 网站 → 添加站点
2. 填写：
   - **域名**：你的域名或服务器 IP
   - **PHP 版本**：纯静态
   - **数据库**：不创建
3. 点击"提交"

### 12.2 配置反向代理

1. 网站列表 → 点击站点名称 → 反向代理 → 添加反向代理
2. 填写：
   - **代理名称**：`blog`
   - **目标URL**：`http://127.0.0.1:3000`
   - **发送域名**：`$host`
3. 点击"提交"

### 12.3 配置 API 代理

1. 网站列表 → 站点设置 → 配置文件
2. 在 `location /` 之前添加：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8765/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_connect_timeout 60s;
    proxy_read_timeout 120s;
    proxy_send_timeout 60s;
}
```

3. 点击"保存"

### 12.4 配置后台管理器代理（推荐）

为避免直接暴露 3001 端口，建议为后台管理器配置子路径代理。在 Nginx 配置文件中继续添加：

```nginx
location /admin-panel/ {
    proxy_pass http://127.0.0.1:3001/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

保存后，通过 `http://你的域名/admin-panel/` 访问后台管理器。

> 如果不需要子路径代理，也可以直接在宝塔面板"安全"中放行 3001 端口，通过 `http://你的服务器IP:3001` 访问。但直接暴露端口有安全风险。

## 13. 开放防火墙端口

### 13.1 宝塔面板放行

1. 宝塔面板 → 安全
2. 放行端口：`80`、`443`
3. 如果需要直接访问后台管理器：放行 `3001`

> ⚠️ 3000 和 8765 端口不需要对外放行，Nginx 反向代理已处理。直接暴露有安全风险。

### 13.2 云服务商安全组放行

登录云服务商控制台，在安全组中添加入站规则：

| 端口 | 协议 | 来源 | 说明 |
|------|------|------|------|
| 80 | TCP | 0.0.0.0/0 | HTTP |
| 443 | TCP | 0.0.0.0/0 | HTTPS |

各云服务商位置：
- **阿里云**：ECS → 安全组 → 配置规则
- **腾讯云**：轻量应用服务器 → 防火墙
- **华为云**：弹性云服务器 → 安全组 → 入方向规则

## 15. 配置域名（可选）

### 15.1 添加 DNS 解析

1. 登录域名服务商（阿里云/腾讯云/Cloudflare）
2. 域名管理 → DNS 解析 → 添加 A 记录
   - **主机记录**：子域名（如 `blog`）
   - **记录类型**：A
   - **记录值**：服务器 IP 地址
3. 等待 DNS 生效（几分钟到几小时）

### 15.2 宝塔面板绑定域名

1. 宝塔面板 → 网站 → 站点设置 → 域名管理
2. 输入域名 → 添加

### 15.3 更新 CORS 配置

域名生效后，更新 `.env` 文件中的 `CMS_ALLOWED_ORIGINS`：

```bash
CMS_ALLOWED_ORIGINS=https://你的域名,https://你的域名/admin-panel/
```

修改后重启后端：

```bash
pm2 restart silentfall-blog-api
```

## 18. 常见错误与解决方案

### 18.1 前端页面空白或样式丢失

**原因**：构建后未复制静态资源到 standalone 目录。

**解决**：
```bash
cd /www/wwwroot/SilentFall_Blog/SFBlogs
cp -r .next/static .next/standalone/.next/
cp -r public .next/standalone/
pm2 restart silentfall-blog-web
```

### 18.2 后端启动失败

**原因**：Python 依赖缺失或 MongoDB 未启动。

**解决**：
```bash
pip3 install fastapi uvicorn python-multipart PyYAML markdown markdownify httpx requests pymongo python-jose[cryptography] bcrypt
mongosh --eval "db.runCommand({ ping: 1 })"
pm2 logs silentfall-blog-api --lines 50
```

### 18.3 构建时内存不足（OOM）

**原因**：服务器内存不足（1GB 以下），Next.js 构建需要较多内存。

**解决**：添加 2GB swap：
```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 18.4 评论或访客说说无法提交

**原因**：MongoDB 连接失败或后端 API 未运行。

**解决**：
```bash
curl http://127.0.0.1:8765/api/status
mongosh --eval "db.runCommand({ ping: 1 })"
pm2 logs silentfall-blog-api --lines 50
```

### 18.5 文章发布后前台不显示

**原因**：ISR 缓存未过期（文章页 60 秒、首页 10 分钟）。

**解决**：等待缓存过期后刷新，或 `pm2 restart silentfall-blog-web`。

### 18.6 端口被占用

**解决**：
```bash
lsof -i :3000
lsof -i :3001
lsof -i :8765
kill -9 <PID>
```

或在启动脚本中修改 `PORT` 环境变量。

### 18.7 Nginx 反向代理 502 错误

**原因**：Nginx 无法连接到后端服务。

**解决**：
```bash
pm2 list                    # 检查服务是否运行
nginx -t                    # 检查 Nginx 配置
systemctl restart nginx     # 重启 Nginx
```

### 18.8 后台管理器登录后 Token 频繁失效

**原因**：未配置 `CMS_SECRET_KEY`，每次重启后端后密钥重新生成。

**解决**：在 `.env` 文件中设置固定的 `CMS_SECRET_KEY`，然后重启后端：
```bash
pm2 restart silentfall-blog-api
```

### 18.9 CORS 跨域错误

**原因**：`CMS_ALLOWED_ORIGINS` 未正确配置。

**解决**：在 `.env` 文件中设置实际的前端域名：
```bash
CMS_ALLOWED_ORIGINS=https://你的域名,https://你的域名/admin-panel/
```

修改后重启后端：`pm2 restart silentfall-blog-api`
